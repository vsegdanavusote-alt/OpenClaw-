Навык работы с GitHub
Для взаимодействия с GitHub используйте ghинтерфейс командной строки (CLI). Всегда указывайте, --repo owner/repoесли вы находитесь не в каталоге Git, или используйте URL-адреса напрямую.

Запросы на слияние (Pull Requests)
Проверить статус запроса на слияние в CI:

gh pr checks 55 --repo owner/repo
Список последних запусков рабочего процесса:

gh run list --repo owner/repo --limit 10
Просмотрите результаты выполнения и узнайте, какие шаги завершились неудачей:

gh run view <run-id> --repo owner/repo
Просматривайте журналы только для неудачных шагов:

gh run view <run-id> --repo owner/repo --log-failed
API для расширенных запросов
Эта gh apiкоманда полезна для доступа к данным, недоступным через другие подкоманды.

Получите PR с указанием конкретных полей:

gh api repos/owner/repo/pulls/55 --jq '.title, .state, .user.login'
Вывод в формате JSON
Большинство команд поддерживают --jsonструктурированный вывод. Вы можете использовать его --jqдля фильтрации:

gh issue list --repo owner/repo --json number,title --jq '.[] | "\(.number): \(.title)"'
